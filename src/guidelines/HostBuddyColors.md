## ❌ COLORS TO NEVER USE

**DO NOT USE** any colors not explicitly listed above, including but not limited to:
- Any Bootstrap default colors
- Any Tailwind CSS default colors (unless they match the approved palette)
- Any Material UI default colors
- Any arbitrary hex values
- Any RGB, HSL, or other color formats unless converted to approved hex values

### Common Violations to Avoid:
- ❌ `#24262E` (Neutral-900) used for **borders** — This color is APPROVED for card backgrounds only. For borders, use `#013280` (Brand-900) instead
- ❌ Random gray shades not in the Neutral palette
- ❌ Random blue shades not in the Brand palette
- ❌ Bootstrap grays like `gray-700`, `gray-800`, etc.
- ❌ Tailwind colors like `blue-500`, `slate-800`, etc.

## Component-Specific Color Guidelines

### Primary Buttons
```tsx
className="bg-[#01255e] border border-[#013280] text-white hover:bg-[#013a8a]"
style={{ boxShadow: '0 0 15px rgba(62, 136, 247, 0.3)' }}
```

### Secondary Buttons
```tsx
className="bg-[#3e88f7] border border-[#013280] text-white hover:bg-[#5296f8]"
```

### Input Fields
```tsx
className="bg-[#0f1117] border border-[#013280] text-white placeholder:text-[#4a5568] focus:border-[#3e88f7]"
style={{ boxShadow: 'inset 0 1px 2px rgba(0, 0, 0, 0.3)' }}
```

### Dropdowns
```tsx
className="bg-[#17191f] border-2 border-[#013280]"
style={{ boxShadow: '0 0 30px rgba(62, 136, 247, 0.2)' }}
```

### Selected/Active States
```tsx
className="bg-[#3e88f7] text-white"
```

### Hover States (for dark backgrounds)
```tsx
className="hover:bg-[#0f1117]"
```

### Success States
```tsx
className="bg-[#10b981] text-white"
```

### Error/Delete States
```tsx
className="bg-[#d4183d] text-white"
```

### Disabled States
```tsx
className="bg-[#4a5568] text-[#a6a9b2] cursor-not-allowed opacity-50"
```

## Migration Guide

When updating components to use the approved palette:

1. **Replace `#24262E` borders** → Use `#013280` (Brand-900). Note: `#24262E` (Neutral-900) IS approved for card backgrounds
2. **Replace generic grays** → Use Neutral palette (0-1100)
3. **Replace custom blues** → Use Brand palette (50-1100)
4. **Replace success colors** → Use Green-500 (`#10B981`)
5. **Replace warning colors** → Use Orange or `#FBBF24`
6. **Replace error colors** → Use Red-700 (`#D4183D`)
7. **Add appropriate glows** → Use Brand-500 based rgba values

## Color Contrast & Accessibility

All color combinations in this palette have been tested for WCAG AAA or AA compliance as indicated in the original Figma design. When pairing colors:

- ✅ White text on Brand-500 or darker
- ✅ White text on Green-500 or darker
- ✅ White text on Neutral-600 or darker
- ✅ Dark text (#013280) on Brand-50 through Brand-300
- ✅ Dark text (#024700) on Green-50 through Green-300
